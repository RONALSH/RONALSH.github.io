2、浮动（float）

文档流

​ 我们所创建的的元素都是在文档流中进行排列

​ 块元素 独占一行 宽度默认是父类的全部 默认高度是被内容撑开

​ 行内元素 只占自身的大小 宽度和高度都是默认被内容撑开

2-1、 脱离文档流之后的变化（重点）

​ 设置浮动之后，脱离文档流

​ 块元素和行内元素都一样了

​ 特点 ： 元素不在独占页面的一行

​ 宽度和高度默认是内容的大小，可以自己设置宽度和高度

float可以让块元素在页面水平排列

2-2、 制作水平布局

float： left

​ 设置浮动之后，块元素完全脱离文档流，在文档流中不在占用位置，其他在文档流中的元素的位置会发生变化

​ 文档流中的元素会被文档流上的元素覆盖

​ 不会从父类元素中移出

2-3、 浮动不会盖住文字

2-4、 高度塌陷

​ 高度塌陷是浮动布局比较常见的问题

​ 在浮动布局中，父元素的高度如果没有被写死，就是默认被子元素撑开的

​ 如果父元素高度没有写死，子元素浮动之后，子元素就会脱离文档流

​ 父元素中就没有内容了，所以高度就没有了

​ 父元素高度丢失，原本下面的元素就会自动上移

2-5、 解决高度塌陷（在父元素开启BFC）

​ 开启BFC该元素可以变成一个独立的布局区域

​ 该元素可以包含浮动的元素，不会塌陷

开启BFC之后的特点：

​ 1、开启BFC的元素不会被浮动元素覆盖，

​ 2、开启BFC的元素子元素和父元素外边距不会覆盖

​ 3、开启BFC的元素可以包含浮动的子元素

开启BFC的方法

​ 1、设置元素浮动（不推荐使用，负作用较大） float ： left

​ 2、把元素设置为行内元素（不推荐使用，负作用较大） display ： inline-block

​ 3、将元素的 overflow 设置为一个非 visible 的值

​ overflow ： hidden

​ overflow ： auto

2-6、 clear（清除浮动给当前元素的影响）

clear作用：

​ 清除浮动元素对当前元素所产生的影响

​ clear ： left 清除左侧浮动元素对当前元素的影响

​ clear ： right 清除右侧浮动元素对当前元素的影响

3、定位（position）

定位：就是将指定的元素摆放到页面的任意位置
通过定位可以任意的摆放元素

3-1、 relative（相对定位）（不脱离文档流）

​ 将 position 属性设置为 relative 则开启元素的相对定位

相对定位特点：

​ 1、如果不设置偏移量，元素无变化

​ 2、相对定位是参照元素原本在文档流中的位置进行定位

​ 3、相对定位会提升元素的层级（会掩盖在文档流中的元素）

​ 4、相对定位不会使元素脱离文档流（不影响其他元素本身的位置（可能会掩盖其他元素））

​ 5、相对定位不会改变元素的性质（块还是块，行内还是行内）

​ 偏移量：

3-2、 absolute（绝对定位）

​ 将 position 属性设置为 absolute 则开启元素的绝对定位

绝对定位特点：

​ 1、如果不设置偏移量，元素无变化

​ 2、脱离文档流，(宽高自己设置，要不然内容多大，宽高就多大)

​ 3、绝对定位会改变元素的性质，行内变成块（脱离文档流都会这样）

​ 4、提升一个层级，会掩盖在文档流中的元素（脱离文档流都会这样）

​ 5、绝对定位是相对于其包含块进行定位的

​ 包含块就是离它最近的开启了定位的祖先元素

​ 祖先元素都没有开启定位，根元素就是包含块

绝对定位的水平布局和垂直布局

​ const 声明方式，除了具有 let 的上述特点外，其还具备一个特点，即 const 定义的变量，

​ 一旦定义后，就不能修改，即 const 声明的为常量。

总结

  1. var 声明的变量属于函数作用域，let 和 const 声明的变量属于块级作用域；
  2. var 存在变量提升现象，而 let 和 const 没有此类现象；
  3. var 变量可以重复声明，而在同一个块级作用域，let 变量不能重新声明，const 变量不能修改。

Vue生命周期

什么是生命周期？

​ 从Vue实例创建、运行、到销毁期间，总是伴随着各种各样的事件，这些事件，统称为生命周期！

​ Vue会在关键时刻帮我们调用一些特殊的函数

创建期间的生命周期函数：
beforeCreate：实例刚在内存中被创建出来，此时，还没有初始化好 data 和 methods 属性

​ created：实例已经在内存中创建OK，此时 data 和 methods 已经创建OK，此时还没有开始 编译模板

​ beforeMount：此时已经完成了模板的编译，但是还没有挂载到页面中

​ mounted：此时，已经将编译好的模板，挂载到了页面指定的容器中显示

  • 运行期间的生命周期函数：
    • beforeUpdate：状态更新之前执行此函数， 此时 data 中的状态值是最新的，但是界面上显示的 数据还是旧的，因为此时还没有开始重新渲染DOM节点
    • updated：实例更新完毕之后调用此函数，此时 data 中的状态值 和 界面上显示的数据，都已经完成了更新，界面已经被重新渲染好了！
  • 销毁期间的生命周期函数：
    • beforeDestroy：实例销毁之前调用。在这一步，实例仍然完全可用。
    • destroyed ：Vue 实例销毁后调用。调用后，Vue 实例指示的所有东西都会解绑定，所有的事件监听器会被移除，所有的子实例也会被销毁。

挂载流程

beforeCreate（数据代理、数据监测创建之前）

​ 数据代理还没开始， 无法访问data中的数据 methods中的方法

​ 页面还没解析好 页面中的数据也没有加载好

created（数据代理、数据监测创建之后）

​ 可以访问data中的数据 methods中的方法

​ 页面还没解析好 页面中的数据也没有加载好

beforeMount（对DOM操作最终都不奏效）

​ 已经完成了模板的编译，但是还没有挂载到页面中

​ 页面呈现的是未经Vue编译的DOM

​ 数据还没加载，HTML中的东西也没解析好

mounted（挂载完毕）

​ 已经将编译好的模板，挂载到了页面指定的容器中显示

​ 页面呈现的是经过Vue编译的DOM

​ 数据加载完成，HTML中的东西也解析好了

更新流程

​ 挂载流程之后，页面已经初次加载完成，

​ 如果点击页面触发事件（比如点击按钮），数据就会改变，就会进入更新流程

​ （数据有更改，就进入更改流程）

beforeUpdate（更新之前）

​ 数据更新了，但是页面还是旧的页面

​ data里面的 n 变成 2 ， 页面上还是 1

updated（更新完毕）

​ 数据是新的，页面也是新的

销毁流程

beforeDestroy（实例销毁之前调用）
                                     实例vm中的 data 、methods、指令都处于可用状态

​ 但是页面不会再改变了（vm已进入销毁流程，不会再回到更新流程来改变页面上的数据了）

destroyed（实例销毁之后调用）

​ vm销毁后，DOM绑定的原生事件还可以调用（控制台还可以显示运行了add），但是数据不会改变

总结

​ 常用的生命周期：

​ 1、mounted：初始化操作（发送ajax请求、启动计时器、绑定自定义事件、订阅消息等）

                     2、beforeDestroy：收尾工作（消除计时器、解绑自定义事件、取消订阅消息等）

​ docker pull mysql:5.7

启动mysql

参数解释：

  • –name 容器名字
  • -p 3308:3306 物理机端口：容器内部端口
  • -e 运行参数 初始化 root 用户的密码
  • -d 后台运行 mysq5.7 镜像名字加标签

-v 目录挂载，**-v /mydata/mysql/log:/var/log/mysql** 表示将 docker里面mysql容器的/var/log/mysql目录挂载到宿主linux系统的 /mydata/mysql/log 目录下，方便查看。
